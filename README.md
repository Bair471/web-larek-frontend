# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Webpack

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- src/pages/index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/index.ts — точка входа приложения
- src/scss/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Установка и запуск
Для установки и запуска проекта необходимо выполнить команды

```
npm install
npm run start
```

или

```
yarn
yarn start
```
## Сборка

```
npm run build
```


или

```
yarn build
```

Класс Component<T>
Абстрактный класс для управления элементами интерфейса и их поведением.

Методы:

toggleClass — переключает класс элемента.

setText — изменяет текстовое содержимое элемента.

setImage — задает изображение и альтернативный текст.

setDisabled — управляет активностью элемента.

setHidden, setVisible — скрывает или показывает элемент.

render — метод для отображения компонента (переопределяется в дочерних классах).

Класс View<T>
Расширяет Component<T>, добавляя взаимодействие с брокером событий.

Класс Modal
Реализует модальное окно, позволяя управлять его состоянием и генерировать события.

Поля:

_closeButton — кнопка закрытия.

_content — область содержимого окна.

Методы:

open — открывает модальное окно и отправляет событие modal:open.

close — закрывает окно, очищает его содержимое и отправляет событие modal:close.

render — обновляет содержимое модального окна и открывает его.

Класс Form
Класс для работы с формами, генерирующий события при изменении данных и отправке формы.

Поля:

_submit — кнопка отправки.

_errors — область отображения ошибок.

Сеттеры:

valid — включает или отключает кнопку отправки.

errors — обновляет текст ошибок.

Методы:

onInputChange — вызывает событие field:change при изменении данных.

render — обновляет отображение формы.

Класс OrderForm
Специализированная форма для оформления заказа.

Поля:

_paymentCard — кнопка выбора оплаты картой.

_paymentCash — кнопка выбора оплаты наличными.

_address — поле ввода адреса.

Сеттеры:

payment — обновляет стили кнопок в зависимости от способа оплаты.

address — задает значение для адреса доставки.

Класс Basket
Управляет содержимым корзины и её состоянием.

Поля:

template — шаблон корзины.

_list — список товаров.

_total — сумма заказа.

_button — кнопка оформления заказа.

Методы:

toggleButton — изменяет активность кнопки заказа.

Сеттеры:

items — обновляет список товаров.

total — обновляет сумму заказа.

Класс Card
Отвечает за отображение товара в каталоге, корзине и модальном окне.

Поля:

_title — заголовок товара.

_image — изображение.

_price — цена.

_category — категория.

_description — описание.

_button — кнопка действий.

Сеттеры:

id — устанавливает идентификатор.

title — задает название.

price — задает цену.

category — задает категорию.

image — устанавливает изображение.

description — задает описание.

button — меняет текст на кнопке.

Класс Contacts
Форма для ввода контактных данных.

Поля:

_email — поле email.

_phone — поле телефона.

Сеттеры:

email — устанавливает email.

phone — задает номер телефона.

Класс EventEmitter
Реализует механизм подписки и отправки событий, позволяя различным частям приложения взаимодействовать между собой.

Поля:

_events — объект, хранящий подписчиков событий.

Методы:

on — подписка на событие.

off — удаление подписки на событие.

emit — отправка события.

onAll — подписка на все события.

offAll — удаление всех подписок.

trigger — возвращает функцию для вызова определенного события.


Класс Page
Главная страница каталога товаров.

Поля:

_counter — счетчик товаров в корзине.

_catalog — каталог товаров.

_wrapper — контейнер страницы.

_basket — кнопка корзины.

Сеттеры:

counter — обновляет счетчик товаров.

catalog — обновляет список товаров.

locked — управляет состоянием страницы (блокировка).

Класс AppData
Хранит и управляет данными приложения.

Поля:

items — список товаров.

preview — товар для предпросмотра.

basket — корзина.

order — данные заказа.

formErrors — ошибки валидации.

Методы:

setItems — обновляет список товаров.

setPreview — задает товар для предпросмотра.

isInBasket — проверяет наличие товара в корзине.

addToBasket — добавляет товар в корзину.

removeFromBasket — удаляет товар из корзины.

clearBasket — очищает корзину.

setPayment — задает способ оплаты.

setOrderField — обновляет данные заказа.

validateOrder — проверяет правильность данных.

clearOrder — очищает данные заказа.

Класс Success
Окно подтверждения оформления заказа.

Поля:

_close — кнопка закрытия.

_total — сумма заказа.

Сеттеры:

total — обновляет сумму заказа.

Класс WebLarekApi
Расширяет Api, добавляя методы для работы с сервером проекта.

Поля:

cdn — сервер контента.

Методы:

getProductList — получает список товаров.

getProduct — получает товар по ID.

createOrder — отправляет заказ.

Презентер
Организует взаимодействие компонентов, управляя событиями.

События приложения
Изменение данных:

items:change — изменение списка товаров.

preview:change — изменение товара в предпросмотре.

basket:change — изменение содержимого корзины.

formErrors:change — обновление ошибок формы.

События интерфейса:

modal:open — открытие модального окна.

modal:close — закрытие модального окна.

basket:open — открытие корзины.

card:select — выбор карточки товара.

order:open — открытие формы заказа.

form:submit — отправка формы.

form.field:change — изменение значения в поле формы.

Основные типы и интерфейсы
IProduct — товар.

IBasket — корзина.

IOrder — заказ.

TOrder — данные заказа и контактов.

TOrderForm — форма заказа.

TContacts — форма контактов.

IOrderResult — результат оформления заказа.
